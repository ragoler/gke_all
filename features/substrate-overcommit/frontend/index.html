GKE Agent Substrate — Overcommit (suspend/resume)
← Return to Hub

Agent sessions sit idle most of the time — so Substrate packs many logical actors onto a small shared worker pool and suspends idle sessions to snapshots, resuming them on demand with in-memory state intact. Below: 10 actor lanes on a 2-worker pool. Each actor is a tiny in-RAM counter — Touch bumps it. Suspend a lane, resume it (or just Touch it — resume-on-connect), and the count picks up exactly where it left off: proof the memory state survived the round-trip while the worker slot was free. With auto-suspend on (default), each lane suspends itself the moment its counter bump completes — the slot frees instantly, so rapid Touches across many lanes show live slot churn on the 2-worker pool. Every button is a real ate-api-server gRPC call.
